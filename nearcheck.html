How NearCheck Lite Works

Our streamlined process makes attendance tracking effortless for both students and teachers while maintaining rigorous security standards.

1

Student Arrival

Students open the NearCheck Lite app when they arrive at school or enter the classroom. The app runs quietly in the background without interrupting their routine.

2

Secure Verification

Our system verifies the student's device location against predefined classroom coordinates using multiple data points for accuracy, including WiFi signatures and Bluetooth beacons.

3

Device Fingerprinting

Each device is anonymously fingerprinted using non-personal identifiers to detect potential sharing while respecting privacy.

4

Teacher Dashboard

Teachers see real-time attendance status with visual indicators for verified students, pending verifications, and any alerts requiring attention.

Frequently Asked Questions

Find answers to common questions about NearCheck Lite implementation and usage.

What is Nearcheck Lite?

Nearcheck Lite is a real-time, smart geolocation-based attendance system designed for educational environments. It allows teachers to manage attendance sessions with precise location verification while prioritizing user privacy and data security. The system features a sleek UI/UX inspired by Apple design principles and physics-based animations for a smooth, intuitive user experience.

What technologies power Nearcheck Lite?

The frontend uses HTML, CSS, JavaScript, and Fontawesome icons for a sleek, responsive UI. The backend is built on Firebase Authentication, Firestore, and Realtime Database for secure, scalable user and attendance data management. Cloud Functions handle backend logic like check-in validation, and Firebase Hosting serves the app. Geolocation relies on the browser's native API, ensuring no paid GPS services are needed.

Is Nearcheck Lite available in multiple languages?

Currently, Nearcheck Lite supports English and Tagalog languages. This ensures accessibility for a wider range of users in relevant educational regions. Additional languages can be added based on demand and localization efforts.

Do I need continuous internet connection for NearCheck Lite to work?

Yes, NearCheck Lite requires a stable internet connection to perform check-ins and access real-time information. Due to browser limitations and its reliance on cloud-based storage, continuous connectivity is necessary for the app to function properly. While some data may be temporarily cached, core features such as attendance tracking and updates depend on an active network connection.

Is there offline access or local caching available?

No. Nearcheck Lite requires a real-time internet connection for all interactions. The app does not cache data or support offline mode to maximize security and ensure that attendance data is always current and verified in real-time.

How is my location data protected and stored?

We take location privacy extremely seriously. Location data is only used momentarily for attendance verification during designated class times. Raw location data is never stored permanently on our servers. The system only retains verification results (present/absent) along with anonymized device identifiers for security purposes. Teachers and administrators only see verification outcomes, not actual location histories.

How does real-time location verification work without storing location history?

The app uses the browser's built-in Geolocation API to verify if a user is within the defined attendance area during check-in. Location data is captured only momentarily at the time of check-in and is securely sent to Firebase for validation. No continuous location tracking or history is stored, ensuring user privacy is maintained while confirming attendance authenticity.

What data is stored in Firebase and how is it secured?

Nearcheck Lite securely stores only essential data in Firebase, including check-in timestamps, geolocation at check-in (not full history), and user IDs. It uses Firebase Authentication to securely manage user identities and Firestore/Realtime Database for storing attendance records. All data transmission is encrypted, and the app does not use local browser storage to prevent unauthorized access or data leakage.

How does Nearcheck Lite ensure privacy and prevent tracking?

The system is designed with a privacy-first philosophy. It only processes the minimal data necessary for check-in sessions, does not track users beyond attendance events, and does not sell data to third parties or display ads. End-to-end encryption and explicit permission requests ensure transparency and control over user data.

How is user security handled in account management?

Account management includes secure password changes (requiring current password and email confirmation), deletion (requiring password and 2FA email confirmation), and limited name/email changes (only within 30 days of registration, secured by password). These measures protect user accounts from unauthorized access and modifications.

What happens if multiple students use the same device?

Our system detects device reuse through anonymous fingerprinting techniques and alerts the teacher. However, we understand some students legitimately share devices due to economic circumstances. Teachers can review these cases individually through the admin panel and approve valid shared device situations. The system learns these patterns over time to reduce false alerts.

Can teachers see historical device associations?

No. For privacy reasons, teachers only see current verified student information associated with devices. Historical device associations from previous semesters or school years are not displayed or stored in identifiable form. This protects student privacy across academic periods while still maintaining current security.

How does the system handle students without smartphones?

We recommend schools maintain alternative procedures for students without personal devices. Options include shared classroom tablets checked out during attendance periods, manual override by teachers for verified cases, or traditional methods for these specific students. Our system is flexible enough to accommodate mixed verification approaches.

How does the Advanced Device Detector (ADD+) work?

ADD+ monitors and flags multiple check-ins originating from the same device for different students on the same day. This helps detect potential misuse or fraudulent attendance. When flagged, teachers receive an alert and can approve or deny the check-in. Denied check-ins mark the student as absent, ensuring attendance integrity.

How does Nearcheck Lite maintain high performance on low-end devices?

The app uses a minimal codebase with optimized assets to ensure fast loading and smooth operation. Physics-based animations and transitions are hardware-accelerated where possible, and the app is designed as a single-page application (SPA) with efficient data fetching and rendering. This approach reduces resource usage and improves responsiveness across device types.

Can students check in automatically when near their teacher?

Yes, with the student's permission, Nearcheck Lite can prompt an automatic check-in when the device detects proximity to the teacher's location. This feature requires temporary or permanent location permission and resets after 24 hours of inactivity to protect privacy and prevent unwanted automatic check-ins.

What happens if a student is outside the location range during check-in?

If a student is outside the defined location proximity during check-in, the app will display a status such as "You are too far." Students can still mark themselves as absent manually. Teachers have control over approving or denying such check-ins, supported by location data where applicable, maintaining fairness and accuracy in attendance.

Can teachers manage students and sections from their dashboard?

Yes. Teachers can create, edit, and delete sections with secure password confirmation. They can invite students via links or QR codes, manage student attendance statuses, and view monthly attendance summaries. The dashboard provides easy access to schedules, total students, and recent attendance reports for efficient class management.

Are there any notifications or alerts sent to teachers and students?

Yes, Nearcheck Lite can send optional notifications via Firebase Cloud Messaging. Teachers receive alerts for suspicious device use, student check-ins, and approvals or denials of flagged attendance. Students receive feedback on their attendance status and reminders for pending check-ins.

What authentication options are available for teachers and students?

Teachers sign up with full name, email, and password, including Multi-Factor Authentication (MFA) for enhanced security, and must be 20+ years old. Students sign up either via invitation link or manual section code, providing their full name, email or phone, and password, confirming they are 13 years or older. Both roles use a shared sign-in form with password recovery options.

How are errors and empty inputs handled during sign-up and check-in?

The app provides clear, user-friendly error messages and tips when required fields are missing or inputs are invalid. Modals guide users to retry or correct their entries, such as invalid invitation links, incorrect passwords, or missing confirmations. This ensures smooth user experience and reduces frustration.

How does Nearcheck Lite handle user privacy regarding name changes and duplicate detection?

The system tracks and displays name change history to teachers for accountability, while also running algorithms to detect duplicate names or suspicious behavior. These measures help maintain data integrity without compromising privacy, as only necessary information is shared and no personal data is sold or exposed outside the app.

What makes the UI/UX design of Nearcheck Lite unique?

Nearcheck Lite combines Apple-inspired clean, minimalistic design with physics-based animations to create an engaging, human-centric experience. The color palette (blue, black, white) supports clarity and professionalism. Smooth transitions and haptic feedback improve user interaction, making the app both aesthetically pleasing and intuitive.
